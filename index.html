git init       : untuk menginisialisasi folder sebagai repo git
git status     : untuk melihat status repo kita
git add        : untuk memasukkan file kita ke staging area (melacak perubahan file)
                        => git add /nama file/
                        => /nama file/ bisa menggunakan . untuk melacak semua file
                        => harus dilakukan sebelum commit
    
git commit     : untuk menyimpan perubahan pada file kita
                        => git commit -m /pesan commit/
                        => /pesan commit/ berisi deskripsi mengenai perubahan yang kita lakukan
    
git log        : melihat commit yang sudah pernah kita lakukan
                        => git log -/angka/
                        => /angka/ berisi jumlah berapa commit terakhir yang ingin ditampilkan
                        => git log -- /nama file/ (untuk melihat commit pada file tertentu)
                        => /nama file/ berisi nama file yang yang ingin dilihat history-nya
                        => git log --all --decorate --oneline --graph (untuk melihat log secara visual)
    
git checkout   : mengembalikan kondisi file di commit tertentu
                        => git checkout /5 digit pertama hash commit/ -- /nama file/
                        => /nama file/ berisi nama file yang ingin di checkout
                        => /nama file/ dapat dikosongi jika ingin checkout semua file
                        => git checkout /nama branch/ untuk berpindah branch
    
git branch     : melihat branch yang ada
                        => git branch /nama branch/ (membuat branch baru)
                        => git branc -d /nama branch/ (menghapus branch)
                        => git branc -D /nama branch/ (menghapus branch dengan paksa)
                        => git branch --merged (melihat branch apa saja yang sudah di merge)

    
git merge      : menggabungkan branch
                        => git merge /nama branch/
                        => /nama branch/ berisi branch yang ingin di merge
    
git clone      : mengkloning file dari github ke lokal
                        => git clone /url repo github/
    
git remote     : untuk melihat remote yang ada pada file kita
                        => git remote --v (untuk melihat remote kita tersambung kemana)
                        => git remote add /nama remote/ /url repo yang ingin di remote/

    
git push       : mengupload file kita ke github
                        => git push -u /nama remote/ /nama branch/ (hanya dilakukan sekali, seterusnya bisa hanya menggunakan git push)
    
git config     : konfigurasi repo kita
                        => git config --list (terdapat akun yang login di git kita)
                        => git config --global user.name /username git/ (untuk login akun)
                        => git config --global user.email /email git/ (untuk login akun)
    
git fetch      : mengecek repo remote kita
git pull       : menyelesaikan merge conflict saat remote
